# Loopsmith — 1-minute demo recording guide

**Record the live URL** (https://loopsmith-sepia.vercel.app) so it's provably deployed.
Window 1440×900, light mode, hide the bookmarks bar. macOS **Cmd+Shift+5** to record, or QuickTime.

## The timing trick
A real run is ~30–45s × 2 runs > 60s. Record raw at ~3 min, then in an editor (iMovie / CapCut /
Descript) **speed the "Thinking…" waits 3–4×**; keep your clicks/typing real-time. Add a voiceover.

## Shot list (target ~60s final)

| Time | On screen | Say |
|------|-----------|-----|
| 0:00–0:08 | `/build` — type: *"Turn my Slack, calls and email into the 2–3 moves that move revenue, and draft the outreach."* | "Every exec wants an AI agent for their workflow. None can architect one." |
| 0:08–0:20 | Click 3–4 suggested chips; the spec / loop takes shape | "You just talk. Loopsmith interviews you and designs the loop — you never see a prompt or the word 'sensor'." |
| 0:20–0:35 | Run #1 streams: ingests signals → decides → drafts actions → **gate holds at 74, 'held for review'** | "It runs live on real signals, grades itself against a quality bar — and refuses to ship its own weak output." |
| 0:35–0:48 | Add a preference (*"always lead with the dollar amount"*); Run #2 → **climbs to 90+**, memory grows, trend chart rises | "Tell it what you want. It applies that now and remembers it forever — the score climbs every run as it learns you." |
| 0:48–0:58 | `/runs` history + **Download repo** button + a flash of `npm test` green (or `/api/health` JSON) | "Done is model-verified — a test suite, a live URL, a rubric it grades against. And you keep a working Claude Code repo." |
| 0:58–1:00 | Live URL on screen | "Loopsmith. The Chief of Staff that builds your Chief of Staff." |

## The two beats judges explicitly want
1. **You directed Claude + it verified itself** → the gate score + the `npm test` / `/api/workflow`
   verifiable exits. Show one for ~2s.
2. **It caught and fixed a failure on its own** → the gate **holding at 74** (weak/un-personalized),
   then **climbing** once it learns your preference. That's the money shot — linger on it.

## Pre-flight checklist (do once before recording)
- [ ] Open the live URL; walk Build → run → refine → run again → /runs → download. No errors.
- [ ] Confirm the gate holds run 1 (sub-80) and run 2 passes (the climb reads clearly).
- [ ] Optional: pre-run twice so the trend chart already shows a climb if you want a snappier cut.
- [ ] Demo in **dry-run** (pasted/fixture signals), not live OAuth connectors — fewer failure points.
- [ ] Zoom browser to ~110% so text is legible at video resolution.
